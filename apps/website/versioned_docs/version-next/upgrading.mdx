# Upgrading to v10

DayPicker v10 is mostly a cleanup release. If your app is already on v9 and
does not use APIs or props deprecated in v9, most of the upgrade is package and
import changes.

The main changes are:

- APIs and props deprecated in v9 were removed.
- `@daypicker/react` is the new preferred package name. It re-exports
  `react-day-picker`, which remains available for compatibility.
- Non-Gregorian calendars moved to separate add-on packages. If you use imports
  such as `react-day-picker/hebrew`, `react-day-picker/hijri`, or
  `react-day-picker/persian`, install the matching add-on package.

## Upgrade checklist

If you are upgrading from v8 or earlier, first follow the
[v9 upgrade guide](/upgrading) and remove the v9 deprecations there.

1. [Upgrade the package](#1-upgrade-the-package): replace `react-day-picker`
   with `@daypicker/react@next` and update imports.
2. [Replace deprecated API](#2-replace-deprecated-api): if you use deprecated
   v9 props or APIs, replace them with the current names.
3. [Update non-Gregorian package imports](#3-update-non-gregorian-package-imports):
   if you use a non-Gregorian calendar, install the matching add-on package and
   update the calendar imports.
4. [Check local DayPicker wrappers](#4-check-local-daypicker-wrappers): if your
   app has a local `Calendar` or `DatePicker` wrapper, update that wrapper
   first, then check the screens that use it.

## 1. Upgrade the package

The recommended v10 package name is `@daypicker/react`. Remove the previous
`react-day-picker` dependency, then install the package under its preferred
name:

```bash npm2yarn
npm remove react-day-picker
npm install @daypicker/react@next
```

The `react-day-picker` package name remains available in v10 for compatibility,
so existing imports can still work. Use `@daypicker/react` for new projects and
upgraded imports.

```diff
- import { DayPicker } from "react-day-picker";
- import "react-day-picker/style.css";
+ import { DayPicker } from "@daypicker/react";
+ import "@daypicker/react/style.css";
```

Locale imports move the same way:

```diff
- import { es } from "react-day-picker/locale";
+ import { es } from "@daypicker/react/locale";
```

## 2. Replace deprecated API

If your app still uses APIs or props deprecated in v9, replace them with the
current names below. The deprecated names were removed in v10.

### Replace removed navigation props

| Removed prop | Use instead |
| --- | --- |
| `fromMonth` | [`startMonth`](./api/react/interfaces/PropsBase.md#startmonth) |
| `fromYear` | [`startMonth`](./api/react/interfaces/PropsBase.md#startmonth) with the first month of that year |
| `toMonth` | [`endMonth`](./api/react/interfaces/PropsBase.md#endmonth) |
| `toYear` | [`endMonth`](./api/react/interfaces/PropsBase.md#endmonth) with the last month of that year |
| `fromDate` | [`hidden`](./api/react/interfaces/PropsBase.md#hidden) with a `before` matcher, and optionally [`startMonth`](./api/react/interfaces/PropsBase.md#startmonth) |
| `toDate` | [`hidden`](./api/react/interfaces/PropsBase.md#hidden) with an `after` matcher, and optionally [`endMonth`](./api/react/interfaces/PropsBase.md#endmonth) |

```diff
<DayPicker
- fromYear={2010}
- toYear={2021}
+ startMonth={new Date(2010, 0)}
+ endMonth={new Date(2021, 11)}
/>
```

```diff
<DayPicker
- fromDate={new Date(2010, 11, 3)}
+ startMonth={new Date(2010, 11)}
+ hidden={{ before: new Date(2010, 11, 3) }}
/>
```

```diff
<DayPicker
- toDate={new Date(2010, 11, 3)}
+ endMonth={new Date(2010, 11)}
+ hidden={{ after: new Date(2010, 11, 3) }}
/>
```

### Replace removed focus and event props

| Removed prop | Use instead |
| --- | --- |
| `initialFocus` | [`autoFocus`](./api/react/interfaces/PropsBase.md#autofocus) |
| `onWeekNumberClick` | A custom [`WeekNumber`](./api/react/functions/WeekNumber.md) component |
| `onDayKeyUp` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |
| `onDayKeyPress` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |
| `onDayPointerEnter` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |
| `onDayPointerLeave` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |
| `onDayTouchCancel` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |
| `onDayTouchEnd` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |
| `onDayTouchMove` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |
| `onDayTouchStart` | A custom [`DayButton`](./api/react/functions/DayButton.md) component |

For week-based selections, follow the
[week selection example](./guides/custom-selections.mdx#week-numbers-and-starts).

For extra DOM event handlers on each day, compose around the default
[`DayButton`](./api/react/functions/DayButton.md) in the
[Custom Components guide](./guides/custom-components.mdx).

### Replace removed aliases

| Removed API | Use instead |
| --- | --- |
| `formatMonthCaption` | [`formatCaption`](./api/react/functions/formatCaption.md) |
| `formatYearCaption` | [`formatYearDropdown`](./api/react/functions/formatYearDropdown.md) |
| `labelDay` | [`labelDayButton`](./api/react/functions/labelDayButton.md) |
| `labelCaption` | [`labelGrid`](./api/react/functions/labelGrid.md) |
| `isMatch` | [`dateMatchModifiers`](./api/react/functions/dateMatchModifiers.md) |
| `isDateInRange` | [`rangeIncludesDate`](./api/react/functions/rangeIncludesDate.md) |

If you already migrated to the v9 names, there is nothing to change here.

### Replace `components.Button`

The deprecated `components.Button` entry was removed in v10.

Use
[`PreviousMonthButton`](./api/react/functions/PreviousMonthButton.md) and
[`NextMonthButton`](./api/react/functions/NextMonthButton.md) instead.

```diff
<DayPicker
  components={{
-   Button: MyButton
+   PreviousMonthButton: MyPreviousButton,
+   NextMonthButton: MyNextButton
  }}
/>
```

### Update `classNames` and `styles` keys

The deprecated compatibility keys were removed from `classNames` and `styles`.
Use only the current keys from
[`UI`](./api/react/enumerations/UI.md),
[`DayFlag`](./api/react/enumerations/DayFlag.md),
[`SelectionState`](./api/react/enumerations/SelectionState.md), and
[`Animation`](./api/react/enumerations/Animation.md).

Common migrations:

| Removed key | Use instead |
| --- | --- |
| `table` | `month_grid` |
| `nav_button` | `button_previous` and `button_next` |
| `day_selected` | `selected` |
| `day_disabled` | `disabled` |

## 3. Update non-Gregorian package imports

The calendar packages are now published separately from the main React package.
Install the add-on package you use alongside `@daypicker/react@next`.

### Persian calendar

The old `jalali` import path now uses the Persian calendar package.

```bash npm2yarn
npm install @daypicker/react@next @daypicker/persian@next
```

| Removed import | Use instead |
| --- | --- |
| `react-day-picker/jalali` | `@daypicker/persian` |
| `react-day-picker/persian` | `@daypicker/persian` |
| `enUSJalali` from `react-day-picker/locale` | `enUSJalali` from `@daypicker/persian` |
| `faIRJalali` from `react-day-picker/locale` | `faIRJalali` from `@daypicker/persian` |

## 4. Check local DayPicker wrappers

A wrapper is a component in your app that renders `DayPicker` with shared
defaults. Apps often use wrappers to keep styles, class names, localization, or
custom components in one place.

For example:

```tsx
import { DayPicker, type DayPickerProps } from "@daypicker/react";

export function Calendar(props: DayPickerProps) {
  return (
    <DayPicker classNames={classNames} components={components} {...props} />
  );
}
```

This is common when using a design system, a copied example, or a calendar
component generated by a UI library.

Update the wrapper first, because every screen that uses it depends on the same
DayPicker props and imports.

| Check for | What to do |
| --- | --- |
| Imports from `react-day-picker` | Move them to `@daypicker/react`. |
| Removed navigation props like `fromMonth`, `toMonth`, `fromYear`, `toYear`, `fromDate`, or `toDate` | Replace them with `startMonth`, `endMonth`, and `hidden` as needed. |
| Removed focus prop `initialFocus` | Use `autoFocus`. |
| Removed event props like `onWeekNumberClick` or `onDayKeyUp` | Move the behavior into a custom `WeekNumber` or `DayButton` component. |
| `components={{ Button: ... }}` | Use `PreviousMonthButton` and `NextMonthButton`. |
| Old `classNames` or `styles` keys like `table`, `nav_button`, `day_selected`, or `day_disabled` | Replace them with the current keys. |
| Non-Gregorian imports like `react-day-picker/hebrew` | Install the matching add-on package and update the import. |
