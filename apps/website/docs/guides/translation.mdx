# Advanced Translations

Most apps don’t need to manually translate DayPicker. Locales from `@daypicker/react/locale` already include translated month names, weekdays, and ARIA labels.

Use this guide only when you need advanced customization, such as custom wording, accessibility copy, or locale-specific formatting tweaks.

## Translation props

| Prop Name    | Type                                              | Description                                |
| ------------ | ------------------------------------------------- | ------------------------------------------ |
| `labels`     | [`Labels`](../api/react/type-aliases/Labels.md)         | Map the ARIA labels used within DayPicker. |
| `formatters` | [`Formatters`](../api/react/type-aliases/Formatters.md) | Map the functions used as date formatters. |
| `dir`        | `rtl` \| `ltr`                                    | Set the text direction.                    |

## Override ARIA labels

Use the `labels` prop to override any DayPicker label. Pass only the labels you want to change.

```tsx
import { DayPicker, defaultLocale } from "@daypicker/react";
import { fr } from "@daypicker/react/locale";

export function CustomLabels() {
  return (
    <DayPicker
      locale={fr}
      labels={{
        // Shorten the next/previous button copy
        labelNext: () => "Mois suivant",
        labelPrevious: () => "Mois précédent",
        // Keep the default logic for other labels
        labelDayButton: defaultLocale.labels?.labelDayButton,
      }}
    />
  );
}
```

### Translatable labels

| Function                                                             | Description                                                                |
| -------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| [`labelDayButton`](../api/react/functions/labelDayButton.md)               | The ARIA label for the day button.                                         |
| [`labelGrid`](../api/react/functions/labelGrid.md)                         | Return an ARIA label for the month grid, announced when entering the grid. |
| [`labelGridcell`](../api/react/functions/labelGridcell.md)                 | The label for the day grid cell when the calendar is not interactive.      |
| [`labelMonthDropdown`](../api/react/functions/labelMonthDropdown.md)       | The ARIA label for the months dropdown.                                    |
| [`labelNav`](../api/react/functions/labelNav.md)                           | The ARIA label for the navigation toolbar.                                 |
| [`labelNext`](../api/react/functions/labelNext.md)                         | The ARIA label for the next month button.                                  |
| [`labelPrevious`](../api/react/functions/labelPrevious.md)                 | The ARIA label for the previous month button.                              |
| [`labelWeekday`](../api/react/functions/labelWeekday.md)                   | The ARIA label for the weekday column header.                              |
| [`labelWeekNumber`](../api/react/functions/labelWeekNumber.md)             | The ARIA label for the week number cell (the first cell in the row).       |
| [`labelWeekNumberHeader`](../api/react/functions/labelWeekNumberHeader.md) | The ARIA label for the week number header element.                         |
| [`labelYearDropdown`](../api/react/functions/labelYearDropdown.md)         | The ARIA label for the years dropdown.                                     |

## Tweak locale data

If you need to adjust date names or formatting, extend `defaultLocale` (or any DayPicker locale) rather than rebuilding everything.

```tsx
import { DayPicker, defaultLocale } from "@daypicker/react";

const locale = {
  ...defaultLocale,
  localize: {
    ...defaultLocale.localize,
    // Example: shorten weekday names
    day: (n, opts) =>
      defaultLocale.localize.day(n, { ...opts, width: "abbreviated" }),
  },
};

<DayPicker locale={locale} />;
```

Keep overrides minimal so you inherit built-in behaviors like translated labels and month/year formatting.

## Custom formatters

Use the `formatters` prop to customize how dates, week numbers, and weekday names are rendered.

```tsx
import { format } from "date-fns";

<DayPicker
  formatters={{
    formatCaption: (date, options) => format(date, "LLLL yyyy", options),
  }}
/>;
```

| Function                                                               | Description                                                     |
| ---------------------------------------------------------------------- | --------------------------------------------------------------- |
| [`formatCaption`](../api/react/functions/formatCaption.md)                   | Format the caption of the month.                                |
| [`formatDay`](../api/react/functions/formatDay.md)                           | Format the day date shown in the day cell.                      |
| [`formatMonthDropdown`](../api/react/functions/formatMonthDropdown.md)       | Format the month number for the dropdown option label.          |
| [`formatWeekNumber`](../api/react/functions/formatWeekNumber.md)             | Format the week number.                                         |
| [`formatWeekNumberHeader`](../api/react/functions/formatWeekNumberHeader.md) | Format the week number header.                                  |
| [`formatWeekdayName`](../api/react/functions/formatWeekdayName.md)           | Format the weekday name to be displayed in the weekdays header. |
| [`formatYearDropdown`](../api/react/functions/formatYearDropdown.md)         | Format the years for the dropdown option label.                 |
